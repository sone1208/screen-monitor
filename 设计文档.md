## 1. 项目概述

### 1.1 产品定位
一款 Windows 桌面端软件使用监控工具，自动跟踪开机状态下各个应用程序的使用时长与使用时间段，帮助用户了解自己的时间分配情况。

### 1.2 设计原则
- **绿色免安装**：一个文件夹，双击即用，删文件夹即卸载
- **零系统侵入**：不写注册表、不注册服务、不开机自启
- **数据自包含**：所有数据文件（数据库、配置）存放在程序所在文件夹内

### 1.3 目标用户
- 希望提升工作效率、了解时间去向的知识工作者
- 需要做时间统计与自我管理的普通电脑用户
- 隐私敏感，不希望软件在系统里留下痕迹的用户

### 1.4 核心功能摘要
| 功能           | 描述                                               |
|----------------|----------------------------------------------------|
| 自动监控       | 后台无感采集当前前台窗口信息                        |
| 使用统计       | 按日/周/月统计各软件使用时长占比及具体时间段        |
| 可视化展示     | 柱状图、饼图、时间线热力图等                        |
| 数据持久化     | 本地 SQLite 存储，隐私安全                          |
| 系统托盘       | 关闭窗口后缩到托盘继续采集                          |

### 3.2 进程模型

```
单个进程 ScreenMonitor.exe
├── 前台窗口存在时：用户可操作 UI
├── 关闭窗口时：不清除进程，缩到系统托盘继续采集
└── 托盘右键"退出"：保存当前会话，结束进程
```

```
程序启动 -> 开始采集 -> 用户关闭窗口 -> 缩小到托盘

                                       |
            用户双击托盘图标 -> 恢复窗口 <- 用户右键退出 -> 程序结束
```

## 4. 详细功能模块

### 4.1 窗口监控模块（核心）

**采集原理：**
1. 通过 user32.dll 的 GetForegroundWindow() 获取当前前台窗口句柄
2. 通过 GetWindowThreadProcessId() 获取所属进程 ID
3. 通过 QueryFullProcessImageName() 获取可执行文件路径
4. 通过 GetWindowText() 获取窗口标题

**采集策略：**
| 策略项         | 取值                         | 说明                                      |
|----------------|------------------------------|-------------------------------------------|
| 轮询间隔       | 1 秒                         | 平衡精度与性能                            |
| 去重策略       | 连续相同记录合并为一条会话    | 减少存储量，提高可读性                     |
| 空闲检测       | 屏保/锁屏/5分钟无操作        | 自动暂停计时，不计入使用时长               |
| 忽略列表       | config.json 可配置            | 忽略系统进程、不想监控的软件               |

### 4.2 空闲检测模块

| 状态           | 检测条件                        | 行为                         |
|----------------|---------------------------------|------------------------------|
| 正常工作       | 前台窗口正常切换                | 正常采集                     |
| 锁屏状态       | 前台窗口为锁屏/登录界面         | 暂停计时                     |
| 离开状态       | 同一窗口大于5分钟无切换         | 标记为"疑似离开"，暂停计时   |
| 睡眠/休眠      | 系统电源事件                    | 恢复后补齐结束时间           |

### 4.3 数据聚合模块

- **实时聚合**：采集层写入时自动计算当前会话时长
- **定时聚合**：每分钟检查是否有长时间未关闭的会话
- **程序退出时**：自动关闭所有未关闭会话，保存最后状态
- **去重合并**：同一进程同一窗口标题的相邻记录合并为单条会话

### 7.1 P/Invoke 核心方法

```csharp
// NativeMethods.cs
[DllImport("user32.dll")]
static extern IntPtr GetForegroundWindow();

[DllImport("user32.dll", SetLastError = true)]
static extern uint GetWindowThreadProcessId(
    IntPtr hWnd, out uint lpdwProcessId);

[DllImport("user32.dll", CharSet = CharSet.Auto)]
static extern int GetWindowText(
    IntPtr hWnd, StringBuilder lpString, int nMaxCount);

[DllImport("kernel32.dll", SetLastError = true)]
static extern bool QueryFullProcessImageName(
    IntPtr hProcess, uint dwFlags,
    StringBuilder lpExeName, ref uint lpdwSize);
```

### 7.2 数据采集主循环

```csharp
// 在 BackgroundService 或 System.Timers.Timer 中执行
private void OnCollectTick(object sender, EventArgs e)
{
    var hWnd = GetForegroundWindow();
    if (hWnd == IntPtr.Zero) return;

    uint processId;
    GetWindowThreadProcessId(hWnd, out processId);
    var process = Process.GetProcessById((int)processId);
    var windowTitle = GetWindowText(hWnd);

    if (IsIgnored(process.ProcessName))
    {
        CloseCurrentSession();
        return;
    }

    if (IsSameSession(processId, windowTitle))
    {
        ExtendCurrentSession();
    }
    else
    {
        SaveAndCreateNewSession(process, windowTitle);
    }
}
```

### 7.3 空闲检测

```csharp
[DllImport("user32.dll")]
static extern bool GetLastInputInfo(ref LASTINPUTINFO plii);

bool IsUserIdle(int idleThresholdSeconds = 300)
{
    var plii = new LASTINPUTINFO();
    plii.cbSize = Marshal.SizeOf(plii);
    GetLastInputInfo(ref plii);

    var idleTime = Environment.TickCount - plii.dwTime;
    return idleTime > idleThresholdSeconds * 1000;
}
```

### 7.4 数据库路径配置

```csharp
// 数据库文件生成在程序所在目录
public class MonitorDbContext : DbContext
{
    protected override void OnConfiguring(DbContextOptionsBuilder options)
    {
        var dbPath = Path.Combine(
            AppDomain.CurrentDomain.BaseDirectory, "data.db");
        options.UseSqlite($"Data Source={dbPath}");
    }
}
```

### 7.5 系统托盘与关闭行为

```csharp
// App.xaml.cs - 重写关闭行为
protected override void OnStartup(StartupEventArgs e)
{
    // 创建系统托盘图标
    var notifyIcon = new NotifyIcon();
    notifyIcon.Icon = new Icon("app.ico");
    notifyIcon.Visible = true;

    // 主窗口关闭时隐藏到托盘而非退出
    MainWindow.Closing += (s, args) =>
    {
        args.Cancel = true;
        MainWindow.Hide();
        notifyIcon.ShowBalloonTip(1000, "监控中",
            "程序已最小化到托盘，后台采集继续运行", ToolTipIcon.Info);
    };

    // 托盘退出按钮才真正关闭
    notifyIcon.ContextMenu = new ContextMenu(new MenuItem[]
    {
        new MenuItem("打开主界面", (s, e) => MainWindow.Show()),
        new MenuItem("退出", (s, e) => { Shutdown(); })
    });
}
```

## 11. 风险与注意事项

| 风险                            | 影响     | 缓解措施                                       |
|--------------------------------|----------|-----------------------------------------------|
| 高精度轮询消耗 CPU              | 中       | 默认 1s 间隔，允许用户调整；空闲时降低频率       |
| 窗口标题含敏感信息              | 中       | 仅在本地存储，不上传；支持一键清除               |
| 反病毒软件误报                  | 中       | 使用标准 Windows API，可签署数字证书             |
